# git-practice
soft eng git workshop

The main objective of this lab was to learn the basic use of Git and GitHub and understand how they help in managing software projects. Before this lab, I had only a basic idea about version control, but this exercise gave me hands-on experience with the complete workflow. First, I created a new public repository on GitHub and added a C++ file to it. Then I cloned the repository to my local computer so that I could work on it from my own system. After cloning, I edited the C++ file by replacing the sample information with my own details. I also learned how to check the status of the repository to see which files had been modified. After reviewing the changes, I staged the updated file, created a commit with a meaningful message, and pushed the changes back to GitHub. Through this process, I became familiar with the basic Git commands that are commonly used in software development. This exercise helped me understand how Git and GitHub work together to keep project files organized and make it easier to track changes during development.

After completing this lab, I gained practical knowledge of using Git and GitHub for version control. I can now create a repository, clone it to my computer, edit project files, and upload the updated version to GitHub without any major difficulty. I also learned how to use important Git commands such as git clone, git status, git diff, git add, git commit, and git push. This exercise helped me understand how every change in a project can be tracked and managed properly. I also realized the importance of writing meaningful commit messages because they make it easier to understand the history of a project. Overall, this lab improved my confidence in using Git and GitHub, and I believe these skills will help me in future programming courses, group assignments, and software development projects where multiple people work on the same code.
